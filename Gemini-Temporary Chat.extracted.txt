        // Cria a pasta se não existir

// Função auxiliar para ler o JSON
async function lerDados() {
    try {
        const data = await fs.readFile(dataPath, 'utf8');
        return JSON.parse(data);
    } catch (error) {
        // Se o arquivo não existir ou estiver vazio, retorna um 
array vazio
        return [];
    }
}
// Função auxiliar para salvar no JSON
async function salvarDados(dados) {
    await fs.writeFile(dataPath, JSON.stringify(dados, null, 2), 
'utf8');
}

Aqui estão as melhores práticas e sugestões arquiteturais para melhorar a qualidade, a
segurança e a experiência do seu projeto:
1. Segurança da Área Administrativa (Autenticação)
No momento, a sua página admin.html e as rotas de POST/DELETE no Node.js estão
abertas. Qualquer pessoa que descobrir a URL pode apagar ou postar notícias.
O que fazer: Proteger o painel de controle.
A solução moderna: Implementar um sistema de login seguro utilizando o Google
Cloud OAuth 2.0. Assim, você valida um Client ID e um Client Secret, garantindo que
apenas contas do Google autorizadas por você consigam acessar o painel e fazer
requisições para a API.
2. Evolução da Camada de Dados
O arquivo JSON é excelente para essa prototipagem rápida e funciona muito bem para
um volume baixo de dados. Porém, em um portal de notícias real, o arquivo JSON pode
corromper se duas requisições tentarem salvar dados no exato mesmo milissegundo.
O que fazer: Preparar o terreno para um banco de dados real quando o projeto
escalar.
A solução moderna: Substituir a leitura do JSON por uma integração com o
Supabase. Ele fornece a robustez de um banco PostgreSQL com uma API super fácil

3. Modernização da Interface (UI) e Componentes Dinâmicos
O design monocromático já traz uma elegância natural, mas podemos aplicar técnicas
mais modernas para não deixá-lo com aspecto de "template padrão".
Glassmorphism: Em vez de usar apenas fundos pretos ou cinzas sólidos (bg-dark),
você pode aplicar efeitos de glassmorphism (fundo translúcido com desfoque) nos
cartões de notícias ou na barra de navegação. Isso cria profundidade.
Widgets Dinâmicos: Portais de esportes ganham muita vida com dados em tempo
real. Implementar via JavaScript componentes como um contador diário (ex: dias
até a próxima partida, ou dias de alguma punição/evento do clube) atrai a atenção
do usuário logo na index.
4. Otimização de Performance e Imagens
O portal permite que o administrador faça upload de fotos. Se você subir uma imagem
de 5MB direto da câmera, o site ﬁcará pesado para quem acessa pelo celular.
O que fazer: Tratar a imagem no backend antes de salvá-la.
A solução moderna: Utilizar uma biblioteca Node.js chamada sharp junto com o
multer. Com poucas linhas de código, você força o servidor a redimensionar a
imagem, convertê-la para o formato WebP (muito mais leve) e só então salvar no
disco.
5. Paginação do Frontend
Atualmente, se você tiver 100 notícias no JSON, o JavaScript vai injetar 100 cards de uma
vez na página inicial, o que cria uma rolagem inﬁnita indesejada e lentidão.
O que fazer: Limitar a exibição e criar páginas (1, 2, 3...).
A solução moderna: Alterar a rota /api/noticias para aceitar parâmetros (ex:
/api/noticias?pagina=1&limite=10), fazendo com que o Node.js devolva apenas
uma "fatia" do array de notícias por vez.

eu consigo exportar toda essa nossa conversa ?
Como um modelo de inteligência artiﬁcial, eu não possuo uma função nativa de "botão
de exportar" dentro da nossa interface que gere um arquivo .pdf ou .docx
automaticamente. No entanto, você pode facilmente salvar todo o histórico desta
colaboração seguindo um destes métodos:
1. Método Copiar e Colar (Mais simples)
Como você organizou o desenvolvimento do projeto em partes modulares, a forma mais
segura de salvar é:
Selecione o texto: Arraste o mouse do início da nossa conversa até aqui.
Cole em um editor: Abra o Google Docs, Word ou Notion e cole todo o conteúdo.
Vantagem: Você mantém a formatação dos blocos de código e a estrutura dos
nossos arquivos, facilitando a consulta futura caso precise ajustar algo.
2. Dica de organização para o seu projeto
Para que você não perca os códigos após copiar e colar, recomendo que você crie um
documento chamado DOCUMENTACAO_DO_PROJETO.md na pasta raiz do seu computador.
Ao colar os códigos lá, você pode usar a formatação de "Markdown" que usamos
aqui (aqueles três acentos graves ``` antes e depois do código).
Isso transformará seu arquivo de texto em uma base de conhecimento técnica que
qualquer editor (como o VS Code) consegue ler com destaque de sintaxe, facilitando
muito sua vida na hora de dar manutenção no site.
